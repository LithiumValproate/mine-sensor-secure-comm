## 4. 研究与分析

### 4.1 协议设计与实现

#### 4.1.1 系统架构

该系统为基于 MQTT over TLS 的矿井通信安全系统，采用三层架构实现井下数据的安全采集、传输与监控：

**底层（传感器节点层）**：部署于采煤工作面、掘进面等场景的传感器节点，负责温度、瓦斯浓度采集，通过 AES-GCM 加密数据后，以 MQTT 客户端身份向中间层发送消息，并支持 LWT 遗嘱消息保障离线状态感知；

**中间层（MQTT Broker层）**：由 Mosquitto Broker 承载，开放 8883 端口、运行 MQTT 协议，通过 **TLS 1.3 双向认证**建立安全通道，作为"枢纽"转发底层与顶层的加密数据；

**顶层（地面中心服务器）**：依次完成"数据接收→解密→验证→告警"流程，并由本地启动器提供 Web 监控模块呈现实时状态、日志和告警。

##### 4.1.2.1 应用层加密模块

**加密工具类**（[crypto_utils.py](src/mine_sensor_secure_comm/crypto_utils.py)）：

```python
def derive_aes128_key(psk_hex: str, sensor_id: str) -> bytes:
    """使用HKDF-SHA256从PSK派生每个传感器独立的AES-128密钥"""
    psk = bytes.fromhex(psk_hex)
    hkdf = HKDF(
        algorithm=hashes.SHA256(),
        length=KEY_SIZE,
        salt=sensor_id.encode('utf-8'),
        info=HKDF_INFO,
    )
    return hkdf.derive(psk)

def encrypt_payload(
        *,
        psk_hex: str,
        sensor_id: str,
        sensor_type: str,
        seq: int,
        timestamp_ms: int,
        plaintext: dict[str, Any],
        boot_random: bytes,
        version: int = 1,
) -> EncryptedPayload:
    """使用AES-128-GCM加密传感器读数"""
    key = derive_aes128_key(psk_hex, sensor_id)
    nonce = make_nonce(boot_random, seq)
    aad = build_aad(version, sensor_id, sensor_type, seq, timestamp_ms)
    encrypted = AESGCM(key).encrypt(nonce, canonical_json(plaintext), aad)
    ciphertext = encrypted[:-GCM_TAG_SIZE]
    tag = encrypted[-GCM_TAG_SIZE:]
    return EncryptedPayload(
        version=version,
        sensor_id=sensor_id,
        sensor_type=sensor_type,
        seq=seq,
        timestamp_ms=timestamp_ms,
        nonce=b64url_encode(nonce),
        ciphertext=b64url_encode(ciphertext),
        tag=b64url_encode(tag),
    )
```

**加密机制特点**：
- 使用 HKDF-SHA256 从主 PSK 派生每个传感器的独立密钥

- AES-GCM 提供机密性和完整性保护

- AAD（附加认证数据）包含版本号、传感器 ID、类型、序列号和时间戳

- Nonce 由 64 位启动随机数 + 32 位序列号组成，确保唯一性

##### 4.1.2.3 防重放攻击机制

**序列号验证**（[replay_guard.py](src/mine_sensor_secure_comm/replay_guard.py)）：

```python
class ReplayGuard:
    """校验时间戳新鲜度和重复序列号"""
    
    def __init__(self, window_ms: int = 300_000) -> None:
        self.window_ms = window_ms  # 5分钟时间窗口
        self._states: dict[str, SensorReplayState] = {}
    
    def check(self, sensor_id: str, seq: int, timestamp_ms: int, receive_time_ms: int) -> str | None:
        # 时间戳新鲜度检查
        if abs(receive_time_ms - timestamp_ms) > self.window_ms:
            return 'timestamp_out_of_window'
        
        state = self._states.setdefault(sensor_id, SensorReplayState())
        self._prune(state, receive_time_ms)
        
        # 序列号重复检查
        if seq in state.recent_seq:
            return 'replay_detected'
        
        # 序列号回退检查
        if seq <= state.last_seq:
            return 'sequence_rollback'
        
        return None
```

**防重放策略**：
- 时间窗口验证：消息时间戳与接收时间差不超过5分钟

- 序列号重复检测：维护最近接收的序列号集合

- 序列号回退检测：拒绝序列号小于已接收最大序列号的消息

##### 4.1.2.4 地面中心处理核心

**消息处理逻辑**（[center_core.py](src/mine_sensor_secure_comm/center_core.py)）：

```python
class GroundCenterCore:
    """供MQTT回调和测试复用的纯校验核心"""
    
    def __init__(
            self,
            *,
            psk_map: dict[str, str],
            sensor_types: dict[str, str],
            thresholds: dict[str, dict[str, float]],
            replay_guard: ReplayGuard | None = None,
    ) -> None:
        self.psk_map = psk_map
        self.sensor_types = sensor_types
        self.thresholds = thresholds
        self.replay_guard = replay_guard or ReplayGuard()
    
    def process_data_message(
            self,
            payload: dict[str, Any],
            *,
            certificate_identity: str | None,
            receive_time_ms: int,
    ) -> ProcessResult:
        sensor_id = str(payload.get('sensor_id', ''))
        sensor_type = str(payload.get('sensor_type', ''))
        
        # 身份验证
        if sensor_id not in self.psk_map or sensor_id not in self.sensor_types:
            return self._reject('unknown_sensor', sensor_id, {'sensor_id': sensor_id})
        
        # 证书身份校验（当前默认运行链路通常拿不到对端证书CN，因此这里只在集成层显式提供时生效）
        if certificate_identity is not None and certificate_identity != sensor_id:
            return self._reject('identity_mismatch', sensor_id, {
                'certificate_identity': certificate_identity,
                'payload_sensor_id': sensor_id,
            })
        
        # 传感器类型校验
        if self.sensor_types[sensor_id] != sensor_type:
            return self._reject('identity_mismatch', sensor_id, {
                'expected_sensor_type': self.sensor_types[sensor_id],
                'payload_sensor_type': sensor_type,
            })
        
        # 重放检测
        replay_error = self.replay_guard.check(sensor_id, seq, timestamp_ms, receive_time_ms)
        if replay_error:
            return self._reject(replay_error, sensor_id, {'seq': seq, 'timestamp_ms': timestamp_ms})
        
        # AES-GCM解密
        plaintext = decrypt_payload(self.psk_map[sensor_id], payload)
        
        # 阈值告警检测
        value = plaintext.get('value')
        if isinstance(value, int | float):
            alert = threshold_alert(sensor_id, sensor_type, float(value), self.thresholds)
            if alert is not None:
                alerts.append(alert)
        
        return ProcessResult(accepted=True, plaintext=plaintext, alerts=alerts)
```

**处理流程**：
1. 传感器身份验证（PSK映射检查）
2. 证书身份校验（可选；默认运行链路通常由Broker负责接入层校验）
3. 传感器类型校验
4. 重放攻击检测（序列号+时间戳）
5. AES-GCM解密与完整性验证
6. 阈值告警检测

```python
def threshold_alert(
        sensor_id: str,
        sensor_type: str,
        value: float,
        thresholds: dict[str, dict[str, float]],
) -> Alert | None:
    sensor_threshold = thresholds.get(sensor_type, {})
    critical = sensor_threshold.get('critical')
    warning = sensor_threshold.get('warning')
    
    if critical is not None and value >= critical:
        return Alert(
            code=f"{sensor_type}_threshold_exceeded",
            sensor_id=sensor_id,
            severity='high',
            message=f"{sensor_type} critical threshold exceeded",
            details={'value': value, 'threshold': critical},
        )
    
    if warning is not None and value >= warning:
        return Alert(
            code=f"{sensor_type}_threshold_warning",
            sensor_id=sensor_id,
            severity='medium',
            message=f"{sensor_type} warning threshold exceeded",
            details={'value': value, 'threshold': warning},
        )
    
    return None
```

#### 4.3.1 TLS双向认证配置问题

**问题描述**：
客户端连接Broker时，TLS握手失败，报证书验证错误

**原因分析**：

1. CA证书未正确配置到客户端
2. 证书链不完整
3. 证书Common Name与服务器地址不匹配

**解决方案**：

```python
# 确保客户端正确加载CA证书
client.tls_set(
    ca_certs='certs/ca.crt',      # 必须指定CA证书
    certfile='certs/sensor.crt',
    keyfile='certs/sensor.key',
    tls_version=ssl.PROTOCOL_TLS_CLIENT,
)
client.tls_insecure_set(False)    # 禁止跳过证书验证
```

**验证方法**：
使用OpenSSL命令验证证书链：

```bash
openssl verify -CAfile ca.crt sensor.crt
```

#### 4.3.2 AES-GCM Nonce唯一性问题

**问题描述**：
在长时间运行后，出现GCM认证失败

**原因分析**：
Nonce重复导致GCM认证失败。如果在同一密钥下使用相同的nonce加密不同消息，攻击者可以恢复明文

**解决方案**：

```python
def make_nonce(boot_random: bytes, seq: int) -> bytes:
    """生成唯一的96位nonce"""
    # 使用64位启动随机数 + 32位序列号
    return boot_random[:8] + seq.to_bytes(4, 'big')

# 每次传感器启动时生成新的随机数
boot_random = os.urandom(8)
```

**设计原则**：

- 每个传感器启动时生成唯一的64位随机数
- 序列号从0开始严格递增
- 96位nonce在传感器生命周期内保证唯一性

#### 4.3.3 防重放攻击问题

**问题描述**：
历史消息可以被恶意攻击者重放，导致系统接收重复数据

**原因分析**：
缺少序列号和时间戳验证机制

**解决方案**：

```python
class ReplayGuard:
    def __init__(self, window_ms: int = 300_000):
        self.window_ms = window_ms  # 5分钟时间窗口
        self._states: dict[str, SensorReplayState] = {}
    
    def check(self, sensor_id: str, seq: int, timestamp_ms: int, receive_time_ms: int) -> str | None:
        # 时间戳新鲜度检查
        if abs(receive_time_ms - timestamp_ms) > self.window_ms:
            return 'timestamp_out_of_window'
        
        # 序列号重复检查
        if seq in self._states[sensor_id].recent_seq:
            return 'replay_detected'
        
        # 序列号回退检查
        if seq <= self._states[sensor_id].last_seq:
            return 'sequence_rollback'
        
        return None
```

**防护机制**：

- 时间窗口验证：消息时间戳与接收时间差不超过5分钟
- 序列号重复检测：维护最近接收的序列号集合（使用LRU缓存）
- 序列号回退检测：拒绝序列号小于已接收最大序列号的消息

#### 4.3.4 传感器断线检测问题

**问题描述**：
传感器意外断开时，地面中心无法及时发现

**原因分析**：
缺少心跳机制和断线通知

**解决方案**：

```python
# 配置LWT遗嘱消息
offline_payload = json.dumps({
    'sensor_id': sensor_id,
    'status': 'offline',
    'reason': 'unexpected_disconnect',
    'timestamp_ms': now_ms(),
})

client.will_set(
    topic=f'mine/{sensor_id}/status',
    payload=offline_payload,
    qos=1,
    retain=True,
)

# 设置KeepAlive时间（60秒）
client.connect(host, port, keepalive=60)

# 发布在线状态
client.publish(
    topic=f'mine/{sensor_id}/status',
    payload=json.dumps({
        'sensor_id': sensor_id,
        'status': 'online',
        'timestamp_ms': now_ms(),
    }),
    qos=1,
    retain=True,
)
```

**工作原理**：

- 传感器连接时设置LWT消息
- 当传感器异常断开时，Broker自动发布LWT消息
- 地面中心监听状态主题，收到offline消息时触发告警

#### 4.3.5 Wireshark抓包验证

**TLS 1.3双向认证握手过程详解**：

系统实际使用 **TLS 1.3** 协议，相比TLS 1.2大幅简化了握手流程，证书信息被加密传输，提供更强的安全性。

**TLS 1.3 握手流程**：

| 阶段 | 帧号 | 消息类型               | 发送方 | 关键内容                                 | 安全意义                     |
| ---- | ---- | ---------------------- | ------ | ---------------------------------------- | ---------------------------- |
| 1    | 4    | Client Hello           | Client | TLS版本、加密套件、ClientRandom、SNI、key_share | 客户端发起握手，协商加密参数 |
| 2    | 6    | Server Hello + CCS + AppData | Server | 加密套件(TLS_AES_256_GCM_SHA384)、ServerRandom、key_share、加密证书链 | 服务器确认参数，切换加密通道 |
| 3    | 8    | CCS + AppData          | Client | Change Cipher Spec、加密客户端证书、Finished | 客户端切换加密通道，完成握手 |

**TLS 1.3技术参数**：

| 参数 | 值 | 说明 |
|------|-----|------|
| TLS版本 | TLS 1.3 | 最新TLS协议版本 |
| 加密套件 | TLS_AES_256_GCM_SHA384 (0x1302) | AES-256-GCM加密，SHA384哈希 |
| 密钥交换 | x25519 | 椭圆曲线Diffie-Hellman |
| 端口 | 8883 | MQTT over TLS标准端口 |
