## 4. 研究与分析

### 4.1 协议设计与实现

#### 4.1.1 系统架构

该系统为基于MQTT over TLS的矿井通信安全系统，采用三层架构实现井下数据的安全采集、传输与监控：

- **底层（传感器节点层）**：部署于采煤工作面、掘进面等场景的5个传感器节点，负责温度、瓦斯浓度采集，通过AES加密数据后，以MQTT客户端身份向中间层发送消息，并支持LWT遗嘱消息保障离线状态感知；
- **中间层（MQTT Broker层）**：由Mosquitto Broker承载，开放8884端口、运行MQTT3.1.1协议，通过**TLS 1.2双向认证**建立安全通道，作为“枢纽”转发底层与顶层的加密数据；
- **顶层（地面中心服务器）**：依次完成“数据接收→解密→验证→存储”流程，最终通过Web监控模块呈现历史/实时数据，支撑矿井安全态势分析。

全链路依托TLS加密与双向认证机制，保障井下敏感数据在传输、处理环节的机密性与完整性，满足矿井安全生产对通信安全的严苛要求。

##### 功能3：AES-CBC/GCM加密切换

**加密工具类**（[center/crypto_utils.py](file:///c:/Users/lenovo/Documents/trae_projects/Computer%20Network/center/crypto_utils.py)）：

```python
class AESCipher:
    def __init__(self, psk: str, mode: str = "CBC"):
        key = hashlib.sha256(psk.encode()).digest()[:16]
        self.key = key
        self.mode = mode.upper()

    def encrypt(self, data: bytes) -> tuple:
        if self.mode == "CBC":
            return self._encrypt_cbc(data)
        elif self.mode == "GCM":
            return self._encrypt_gcm(data)

    def _encrypt_cbc(self, data: bytes) -> tuple:
        """AES-CBC模式加密"""
        iv = os.urandom(16)
        cipher = Cipher(algorithms.AES(self.key), modes.CBC(iv), backend=default_backend())
        encryptor = cipher.encryptor()
        pad_length = 16 - (len(data) % 16)
        padded = data + bytes([pad_length] * pad_length)
        ciphertext = encryptor.update(padded) + encryptor.finalize()
        return iv, ciphertext

    def _encrypt_gcm(self, data: bytes) -> tuple:
        """AES-GCM模式加密（含认证标签）"""
        nonce = os.urandom(12)
        cipher = Cipher(algorithms.AES(self.key), modes.GCM(nonce), backend=default_backend())
        encryptor = cipher.encryptor()
        ciphertext = encryptor.update(data) + encryptor.finalize()
        return nonce, ciphertext, encryptor.tag

    def _decrypt_gcm(self, nonce: bytes, ciphertext: bytes, tag: bytes) -> bytes:
        """AES-GCM模式解密（自动验证完整性）"""
        cipher = Cipher(algorithms.AES(self.key), modes.GCM(nonce, tag), backend=default_backend())
        decryptor = cipher.decryptor()
        return decryptor.update(ciphertext) + decryptor.finalize()
```

**统一加密接口**：

```python
def encrypt_message(mode: str, data: bytes, psk: str) -> bytes:
    """统一加密接口，支持CBC和GCM模式"""
    cipher = AESCipher(psk, mode)
    if mode == "GCM":
        nonce, ciphertext, tag = cipher.encrypt_with_ad(data)
        return nonce + tag + ciphertext  # GCM格式: nonce(12) + tag(16) + ciphertext
    else:
        iv, ciphertext = cipher.encrypt(data)
        return iv + ciphertext  # CBC格式: iv(16) + ciphertext
```

```python
def create_lwt_message(self):
    """创建遗嘱消息负载"""
    lwt_data = {
        "sensor_id": self.sensor_id,
        "status": "offline",
        "timestamp": int(time.time()),
        "reason": "abnormal_disconnect"
    }
    lwt_json = json.dumps(lwt_data).encode('utf-8')
    
    if self.encryption_mode == "GCM":
        return encrypt_message("GCM", lwt_json, PSK)
    else:
        iv, ciphertext = self.cipher.encrypt(lwt_json)
        return iv + ciphertext
```

```python
def pack_message(sensor_id: str, sequence_num: int, temperature: float, gas: float) -> bytes:
    """打包传感器消息"""
    timestamp = int(time.time())
    sensor_id_bytes = sensor_id.encode('utf-8')[:16]
    sensor_id_bytes = sensor_id_bytes + b'\x00' * (16 - len(sensor_id_bytes))
    return struct.pack('!16sIIdd', sensor_id_bytes, sequence_num, timestamp, temperature, gas)

def unpack_message(data: bytes) -> tuple:
    """解包传感器消息"""
    try:
        sensor_id_bytes, sequence_num, timestamp, temperature, gas = struct.unpack('!16sIIdd', data)
        sensor_id = sensor_id_bytes.decode('utf-8').rstrip('\x00')
        return sensor_id, sequence_num, timestamp, temperature, gas
    except struct.error:
        return None
```

### 4.3 问题分析与解决

#### 4.3.1 遇到的技术难点

**问题1：TLS双向认证配置**
- **问题描述**：客户端与服务端证书验证失败
- **原因分析**：证书链不完整，CA证书未正确配置
- **解决方案**：
  1. 使用OpenSSL生成完整证书链
  2. 配置Mosquitto要求客户端证书验证
  3. 客户端正确加载CA证书

**问题2：加密模式切换**
- **问题描述**：CBC与GCM模式切换时出现解密失败
- **原因分析**：两种模式的消息格式不同（IV长度、认证标签）
- **解决方案**：
  1. 在消息中标识加密模式
  2. 实现统一的加密/解密接口
  3. 通过配置文件指定加密模式

**问题3：防重放攻击**
- **问题描述**：历史消息可以被重放
- **原因分析**：缺少序列号和时间戳验证
- **解决方案**：
  1. 每条消息添加32位序列号
  2. 添加时间戳并验证时间窗口（5分钟）
  3. 维护已接收序列号缓存

**问题4：传感器断线检测**
- **问题描述**：传感器意外断开时无法及时发现
- **原因分析**：缺少心跳机制和断线通知
- **解决方案**：
  1. 使用MQTT遗嘱消息(LWT)机制
  2. 设置KeepAlive时间
  3. 地面中心监听LWT消息并告警

#### 4.3.2 Wireshark抓包验证

**TLS握手过程**：

```
1. Client Hello → Server
   - TLS版本：TLS 1.2
   - 加密套件：TLS_ECDHE_RSA_WITH_AES_256_GCM_SHA384
   - 随机数：ClientRandom

2. Server Hello → Client
   - 选择加密套件
   - 服务器证书（server.crt）
   - 随机数：ServerRandom
   - CertificateRequest（要求客户端证书）

3. Certificate → Server
   - 客户端证书（client.crt）

4. ClientKeyExchange → Server
   - Premaster Secret（用服务器公钥加密）

5. CertificateVerify → Server
   - 客户端签名验证

6. Finished → Server
   - 握手完成

7. 应用数据传输（加密）
```

**抓包截图分析**：
- 第1-6步：TLS握手（约150ms）
- 第7步：加密数据传输（每帧约3ms）
- 应用层数据被完全加密，无法直接读取

## 5. 总结

### 5.1 成果总结

**实现的功能**：

1. **多传感器节点模拟**：支持5个独立传感器节点，每个节点有独立ID、位置和数据采集间隔

6. **瓦斯超限告警**：瓦斯浓度超过1.0%时触发告警

7. **Web实时监控**：实时显示传感器数据、日志详情、加密模式对比
